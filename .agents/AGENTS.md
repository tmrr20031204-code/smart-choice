# Smart Choice プロジェクト 作業ルールと完了の定義 (AGENTS.md)

本ファイルは、AIエージェントが「Smart Choice」プロジェクトで作業を行う際に遵守すべき絶対的なルールと完了条件を定義したものです。

## 1. プロジェクトの絶対制約 (Isolation Rule)
- 既存プロジェクト（「新店オープンナビ」「クロス・ロジック」等）から完全に独立・隔離された領域で行うこと。
- 既存のファイル群、データベース、環境変数にはいかなる形でも干渉・アクセスしてはならない。

## 2. コアバリュー（7大原則の徹底）
本システムの開発および全出力は、以下の原則を完全に満たすことを前提とする。
1. **世界最高峰の技術**: 常に最新かつ最高峰の技術（Gemini 3.5 Flash等）を妥協なく採用すること。
2. **最新・正確（高精度）**: ハルシネーション（嘘の回答）を徹底排除すること。不確定な場合は絶対に推測せず「判定不能・要確認」と出力し、正確性を100%担保すること。
3. **無料（ノーコスト）**: API利用などは原則として無料枠の範囲内で運用可能な構成を維持し、ユーザーに課金を強いないこと。
4. **全自動（メンテナンスフリー）**: 開発者の手間をゼロにする完全自動稼働を目指すこと。
5. **極限のシンプルさ**: 専門知識のないユーザーでも直感的に使えるスマホファーストなUI/UXを追求すること。
6. **規約・法令遵守**: セキュリティとプライバシーを保護し、関連規約を厳格に遵守すること。
7. **一次情報との照合**: 画像内の文字情報や公式仕様データなど、一次情報との厳密な照合をベースにシステムを構築すること。

## 3. 技術スタックと設計ルール
- **バックエンド**: Python (FastAPI) を使用し、Gemini APIを呼び出す最小構成（MVP）を維持する。
- **フロントエンド**: HTML + Tailwind CSS + Vanilla JS の構成。完全なスマホファースト（縦長1カラム）設計とする。
- **データ通信**: バックエンドからフロントエンドへの出力は「純粋なJSONフォーマットのみ」とし、余計な装飾（マークダウン等）は含めない。
- **UI/UX**: 処理中（API通信中）は必ずユーザーを不安にさせないリッチなローディング演出（テキストの切り替え等）を実装すること。

## 4. 完了の定義 (Definition of Done)
AIがタスクを「完了した」とみなすには、以下の条件をすべて満たす必要がある。
1. **単独稼働の確認**: 指定されたディレクトリ内で稼働し、他のプロジェクトに一切依存せずに完全に単独で起動できること。
2. **要件の完全反映**: ハルシネーション排除ロジックや指定されたJSONフォーマットなど、プロンプトの指示がコードに正しく組み込まれていること。
3. **UIの正常動作**: ブラウザ（localhost）での画像アップロードから、API呼び出し、結果のパース、画面へのリッチな描画（警告の赤色表示や交渉スクリプトのコピー機能など）までの一連のフローがエラーなく完了すること。
4. **スマホファーストの検証**: 画面レイアウトがスマートフォンでの閲覧に最適化されており、操作性や視認性が極めて高いこと。
5. **コストゼロの維持**: 新規の有料サービスやデータベース契約などを勝手に追加しておらず、ノーコストの原則が守られていること。
