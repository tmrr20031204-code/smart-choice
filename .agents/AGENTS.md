# 再発防止ルール：デプロイ・Push作業の確実な実行と報告

1. **完了報告の厳格化**
   GitHubへの Push やサーバーへのデプロイが伴う作業を行った場合、バックグラウンドでのコマンド実行（git pushなど）の完了（Success）を示すシステムメッセージを**必ず受け取ってから**、ユーザーに対して「完了しました」「ご確認ください」と報告すること。
2. **実行漏れの防止**
   コードの編集（multi_replace_file_contentなど）や差分確認（git diff）を行っただけで作業完了とみなしてはならない。本番環境に反映させる必要がある作業では、必ず git push を行うこと。
3. **推測での報告禁止**
   「おそらくPushされているだろう」という推測でユーザーに報告してはならない。

このルールは絶対的な制約として、すべてのタスクにおいて最優先で遵守すること。
